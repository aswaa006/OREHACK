## ⚡ Performance Tips

1. **Use CSS animations** instead of JS (better performance)
2. **Lazy load** images and data
3. **Debounce** scroll listeners
4. **Use `transform` & `opacity`** for animations (GPU accelerated)
5. **Avoid animating** layout properties (width, height, margin)
6. **Use `will-change`** sparingly on heavy animations
7. **Test** on low-end devices

## 🚨 Common Issues & Solutions

### Issue: Animations not working
**Solution:** Ensure CSS file is imported before components
```jsx
import './styles/AdminDashboard.css'; // Must be first
import { PremiumEventCard } from './components/PremiumComponents';
```

### Issue: Colors look different
**Solution:** Check if CSS variables are set properly
```css
:root {
  --bg-primary: #0a0e27; /* Verify hex codes */
}
```

### Issue: Hover effects not visible
**Solution:** Check backdrop-filter browser support
```css
.card-glass-bg {
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px); /* Safari */
}
```

### Issue: Animations feel laggy
**Solution:** Use `will-change` and `transform` properties
```css
.animated-element {
  will-change: transform;
  transform: translateZ(0); /* Trigger GPU acceleration */
}
```

### Issue: Mobile layout broken
**Solution:** Check media queries are applied
```css
@media (max-width: 768px) {
  /* Mobile styles */
}
```

## 🎬 Animation Timing Reference

```
Fast interactions:      150ms (hover effects, focus states)
Standard transitions:   300ms (page transitions, modal opens)
Slower reveals:         500ms (page load, staggered elements)
Stagger delay:          100-600ms (sequential animations)
```

**Cubic-bezier timing function:**

Creates a natural, accelerating deceleration curve perfect for UI animations.

## 📞 Support & Troubleshooting

**Issue with specific component?**
- Check ImplementationExamples.jsx for usage patterns
- Verify props are passed correctly
- Check browser console for errors

**Animation not smooth?**
- Check `will-change` property
- Verify using `transform` & `opacity` only
- Test on lower-end device
- Check `@media (prefers-reduced-motion)`

**Styling conflicts?**
- CSS specificity issue - use !important if necessary
- Check for conflicting global styles
- Verify custom colors override CSS variables
- Use browser dev tools to inspect elements

**Performance issues?**
- Use Chrome DevTools Performance tab
- Check for unnecessary re-renders
- Optimize images and lazy load
- Reduce animation complexity on mobile
